텐서플로우 프론트엔드¶

텐서플로우 프론트엔드는 텐서플로우 모델을 TVM으로 가져오는 작업을 돕습니다.

지원되는 버젼:

  • 1.12 이상

확인된 모델:

  • Inception (V1/V2/V3/V4)

  • Resnet (All)

  • Mobilenet (V1/V2 All)

  • Vgg (16/19)

  • BERT (Base/3-layer)

추론을 위한 모델 준비하기¶

불필요한 노드 제거¶

내보내기 프로세스로 추론에 불필요한 많은 노드들이 제거될 것이지만, 유감스럽게도 몇몇 노드는 여전히 남아있을 겁니다. 이렇게 손수 제거해야 하는 노드들은 :

  • Dropout, Dropout 과 DropoutWrapper 포함

  • Assert

None 차원을 상수로 치환¶

TVM은 동적 변화 텐서에 대해서는 최저한으로만 지원합니다. None 차원은 상수들로 교체되어야 합니다. 예컨대 어떤 모델이 (None,20) 모양의 입력을 받을 수 있다면, 그 입력은 (1,20) 와 같은 모양으로 치환되어야 합니다. 그에 따라 모델 역시 그래프 전체에서 동작에 문제가 없도록 수정되어야 합니다.

내보내기¶

텐서플로우 프론트엔드는 유효한 입력 포맷으로서 동결(frozen)된 protobuf (.pb) 나 저장된 모델을 기대하며, 현재로서는 체크포인트(.ckpt)는 지원하지 않습니다. 텐서플로우 프론트엔드가 필요로 하는 graphdef는 액티브 세션이나 TFParser 헬퍼 클래스를 사용해서 추출할 수 있습니다.

모델은 추론을 위해 쓰일 수 있도록 변형을 거쳐 내보내기가 되어야 합니다. "the model for inference. It is also important to set `add_shapes=True` 로 설정하는 것도 중요한데, as this will embed the output shapes of each node into the graph. Here is one function to export a model as a protobuf given a session:

import tensorflow as tf
from tensorflow.tools.graph_transforms import TransformGraph

def export_pb(session):
    with tf.gfile.GFile("myexportedmodel.pb", "wb") as f:
        inputs = ["myinput1", "myinput2"] # replace with your input names
        outputs = ["myoutput1"] # replace with your output names
        graph_def = session.graph.as_graph_def(add_shapes=True)
        graph_def = tf.graph.util.convert_variables_to_constants(session, graph_def, outputs)
        graph_def = TransformGraph(
            graph_def,
            inputs,
            outputs,
            [
                "remove_nodes(op=Identity, op=CheckNumerics, op=StopGradient)",
                "sort_by_execution_order", # sort by execution order after each transform to ensure correct node ordering
                "remove_attribute(attribute_name=_XlaSeparateCompiledGradients)",
                "remove_attribute(attribute_name=_XlaCompile)",
                "remove_attribute(attribute_name=_XlaScope)",
                "sort_by_execution_order",
                "remove_device",
                "sort_by_execution_order",
                "fold_batch_norms",
                "sort_by_execution_order",
                "fold_old_batch_norms",
                "sort_by_execution_order"
            ]
        )
        f.write(graph_def.SerializeToString())

또다른 방법은 `내보내기와 그래프 동결<https://github.com/tensorflow/models/tree/master/research/slim#exporting-the-inference-graph>`_ 입니다.

모델 불러들이기¶

명시적인 모양:¶

To ensure shapes can be known throughout the entire graph, pass the `shape` argument to `from_tensorflow`. This dictionary maps input names to input shapes. Please refer to these test cases for examples.

데이터 레이아웃¶

대부분의 텐서플로우 모델은 NHWC 레이아웃으로 배포됩니다. NCHW 레이아웃은 특히 GPU 상에서 더 나은 성능을 보여준다고 알려져 있습니다. 텐서플로우 프론트엔드는 `layout='NCHW'` 와 `from_tensorflow` 인자를 넘겨 받으면 모델의 데이터 레이아웃을 자동으로 변형할 수 있습니다.

모범 사례¶

  • 동적 모양 대신 정적 모양 텐서를 사용해 주세요 (`None` 차원은 제거해 주세요)

  • 다이내믹 RNN 대신 정적 RNN을 사용해 주세요. `TensorArray` 는 아직 미지원입니다.
